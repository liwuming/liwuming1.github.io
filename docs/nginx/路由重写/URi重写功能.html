URi重写功能

rewrite模块即ngx_http_rewrite_module模块，主要功能是改写请求URI，是Nginx默认安装的模块。rewrite模块会根据PCRE正则匹配重写URI，然后发起内部跳转再匹配location，或者直接做30x重定向返回客户端。 指令执行顺序

首先顺序执行server块中的rewrite模块指令，得到rewrite后的请求URI然后循环执行如下指令

如果没有遇到中断循环标志，此循环最多执行10次，但是我们可以使用break指令来中断rewrite后的新一轮的循环

(1). 依据rewrite后的请求URI，匹配定义的 location 块

(2). 顺序执行匹配到的 location 中的rewrite模块指令

rewrite指令

语法格式 rewrite parttern replacement flag

parttern，perl正则表达式

flag标记说明 break，若rewrite指令在server块，还会继续匹配重定向后的location，若rewrite指令在location块，则会停止， last，本条规则匹配完之后，继续向下匹配新的location uri规则 redirect，302，临时重定向， permanent，301，永久重定向，

rewrite指令可用在server,location块及if语句中

url重写需要一定的正则表达式基础

信息

301永久重定向和302临时重定向的目的是一致的--重定向，所谓重定向就是改变浏览器地址栏的url，然后发出请求

301和302的区别在于seo优化，搜索引擎对301和302的处理方式不同 如果一个网页被收录，后来被网址变动，使用301，搜索引擎会更改久的内容和连接， 对于302，则旧的不会被删除，会增加新的页面连接和内容，但此操作有移动权重的嫌疑，搜索引擎不喜欢302，会降低页面权重，

Google一直强调使用301，而不要用302。

无论301，还是302，浏览器地址栏都会显示重定向后的url

rewrite_log指令

功能描述配置是否开启url重写日志输出功能
语法说明rewrite_log on
默认值off
位置信息http,server,location,if

在将rewrite_log设置为on之后，重写日志将以notice级别输出到error_log中；

rewrite的应用场景

url重写​

隐藏index.php,将动态url伪装成静态url

rewrite ^/index.php/(.*?)$ /$1 last;

注意，这里的flag最好是last，

域名跳转​

# www.ibook.com
server{
    listen 80;
    server_name www.ibook.com;
    
    default_type text/plain;
    location / {
        return 200 'www.ibook.com';
    }
    error_page 404 /404.html;
}


# www.ibook1.com
server{
    listen 80;
    server_name www.ibook1.com;
    
    location / {
        return 301 http://www.ibook.com;
    }
    error_page 404 /404.html;
}

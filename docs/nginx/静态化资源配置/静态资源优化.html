静态资源优化

nginx的gzip_module模块提供了对资源进行压缩处理，该模块会在nginx安装的时候内置到nginx的安装环境中，也就说可以直接使用这些指令

关于nginx的gzip，可以分为两类

  • 静态压缩，使用现成的同名的扩展名为.gz的预压缩文件
  • 动态压缩，

很明显，静态压缩会比动态压缩效率更高一些，但是并不是每个文件都有.gz预压缩文件的，所以在生产环境中通常是静态压缩与动态压缩结合使用的

静态压缩

开启静态压缩很简单，只需要如下一行配置即可搞定

gzip_static  on;

虽然配置简单，但静态压缩需要使用ngx_http_gzip_static_module模块，该模块在默认安装时是不包含在内的，可通过nginx -V命令来查看编译信息，确认是否在编译时加入了该模块，若没有，则需要添加--with-http_gzip_static_module重新编译安装

cd /usr/local/src/nginx-1.22.0
make clean
./configure --sbin-path=/usr/local/sbin/nginx --pid-path=/usr/local/nginx/nginx.pid --conf-path=/etc/nginx.conf --with-http_ssl_module --with-http_gzip_static_module

make 
cp objs/nginx

gzip

ngx_http_gzip_static_module 模块允许发送扩展名为 .gz 的预压缩文件，而不是常规文件。

动态压缩

gzip指令​

设置开启或关闭gzip功能,默认值：off;

语法：gzip on|off;

gzip_types​

指定进行gzip压缩的MIME类型，默认仅对text/html进行压缩，多个值之间用空格分隔

gzip_comp_level​

压缩级别，可选值1-9，1级别最低，9级别最高，一般选择6

gzip_vary:用于在开启gzip压缩时，是否携带vary:accept-encoding;头信息 语法：gzip_vary on|off; 默认值off

gzip_buffers num size:

处理压缩缓存区的个数和大小； gzip_min_length,针对传输数据大小压缩

默认20b，10k

content_min_length file_size

文件大于file_size阈值时，才进行压缩处理

温馨提示

file_size默认单位为字节，也可以使用k,m单位

gzip_proxied​

对代理的结果是否压缩，默认值off；

gzip_proxied指令​

设置当Nginx作为反向代理的时候是否启用动态压缩

根据某些请求和应答来决定是否在对代理请求的应答启用gzip压缩，是否压缩取决于请求头中的“Via”字段，指令中可以同时指定多个不同的参数，意义如下： expired - 启用压缩，如果header头中包含 "Expires" 头信息 no-cache - 启用压缩，如果header头中包含 "Cache-Control:no-cache" 头信息 no-store - 启用压缩，如果header头中包含 "Cache-Control:no-store" 头信息 private - 启用压缩，如果header头中包含 "Cache-Control:private" 头信息 no_last_modified - 启用压缩,如果header头中不包含 "Last-Modified" 头信息 no_etag - 启用压缩 ,如果header头中不包含 "ETag" 头信息 auth - 启用压缩 , 如果header头中包含 "Authorization" 头信息 any - 无条件启用压缩

sendfile on;

位置，http，server,location 通常设置在http块

默认配置下，Nginx 读取本地文件后，在进行网络传输时会先将硬盘文件从硬盘中读取到 Nginx 的文件缓冲区中，操作流程为硬盘 → 内核文件缓冲区 → 应用缓冲区。然后将 Nginx 文件缓冲区的数据写入网络接口，操作流程：应用缓冲区 → 内核网络缓冲区 → 网络接口。

tcp_nopush on;

主要用来提升网络包的传输效率，但必须必须在sendfile打开的状态下才会生效，

tcp_nodelay on;

主要用来提升网络包的实时性，但必须必须在keep-alive打开的状态下才会生效，

http{

sendfile on;
tcp_nopush on;
tcp_nodelay on;

}

由于gzip的配置比较单一，可以单独成文件，对于需要进行压缩的虚拟主机，使用include命令引入即可

cat > /etc/nginx/gzip.conf <<-EOF

gzip on;
gzip_types text/html text/css application/javascript image/jpeg image/png;
gizp_comp_level 6;
gzip_vary on;
#gzip_buffers
gzip_min_length 10k;
gzip_proxied any;

EOF

关于gzip的两个问题

gzip和sendfile的冲突解决

sendfile由内核缓冲区直接给网卡，发送给客户端的，不再 解决方案，在磁盘上的文件直接压缩

浏览器不支持gzip解决

gzip_static指令

检查是否有同名的.gz文件

gzip_static on;

gzip_static来自ngx_http_gzip_static_module模块，但是在nginx安装时默认是没有安装该模块的；

如何添加新模块

--width-http_gzip_static_module

对于nginx的gzip的使用，通常是静态压缩与动态压缩的结合使用，优先使用静态压缩，在预压缩文件不存在的情况下，使用动态压缩；

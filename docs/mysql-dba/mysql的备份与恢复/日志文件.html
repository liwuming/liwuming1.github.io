2. 慢查询日志

配置方式

[mysqld]
# 开启或关闭
slow_query_log=1
# 慢查询阈值，单位毫秒，1秒=1000毫秒
long_query_time=0
# 将没有走索引的语句也记录到slow中
log_queries_not_using_indexes
#将慢查询记录到mysql.slow_log表中
log_output=table

查看mysql的慢查询状态

mysql> show variables like 'slow%';
+---------------------+------------------------------------+
| Variable_name       | Value                              |
+---------------------+------------------------------------+
| slow_launch_time    | 2                                  |
| slow_query_log      | OFF                                |
| slow_query_log_file | /liwuming/mysql/localhost-slow.log |

3 rows in set (0.00 sec)

slow_query_log：慢查询开启状态 slow_query_log_file：慢查询日志存放的位置（一般设置为 MySQL 的数据存放目录）

binlog日志

二进制日志记录了数据变更类--insert和update及delete的事件，由于二进制日志完整的记录了数据的变更记录，因此可用来复制和备份

对于数据未变更类操作，如select，show不会记录， 虽然记录二进制日志，会稍微有些性能的下降，但是可用来复制和备份恢复，一点点的性能损失也是值得的

但是默认情况下，二进制日志并没哟开启，需要配置开启

server=100
log_bin=mysql-binlog
binlog-format=row

如果在log-bin的配置项中增加了扩展名，如bin-log=mysql-binlog.log，扩展名会被忽略掉

关于binlog记录的内容​

内容以事件为最小记录单元，每一条SQL就是一个事件，

查看事件列表 show binlog events,默认查看的就是当前所使用的binlog文件中的事件列表，假设当前所使用的binlog文件为mysql-binlog.000001，show binlog events[ in 'mysql-binlog.000001']

注意：mysql-binlog.000001是需要添加引号的，否则会报错

show binlog logs;

查看binlog列表

show binary logs;

查看正在使用的binlog

show master status;

查看binlog事件

show binlog events [in "binlog_file"];

若查看当前正在使用的binlog的事件列表可省略in binlog_file，binlog_file的引号不可为空

产生一个新的binlog

flush logs;

查看binlog内容

既然是binlog，肯定不能用vi打开，所以需要专门的工具查看，mysqlbinlog

mysqlbinlog binlog_path; mysqlbinlog --base64-output=decode-rows -vvv mysql-binlog

DDL，DML，DCL都会被记录，但是去除select，show等不会改变数据库的语句

对于DCL，DDL,无论是statement模式，还是row模式，均以语句方式记录

对于DML语句，有记录格式可选 row模式， statement模式， mixed模式，

如何选择 row模式，即基于行的日志记录，binlog日志体积会相比statement会比较大，但是对于 statement模式，可读性强，体积更小，但是可能会出现一些问题，

一般为了数据的一致性，选择row模式

设置二进制日志记录格式​

binlog内容记录默认有三种，row模式，max模式

row模式​

row模式，即基于行的日志记录，若一条SQL更新了100个记录行，则row模式需要记录100个行的变化，而s一行就搞定

row，安全性有保障

statement模式，记录哪些insert，update，delete语句本身，​

优点，简单，日志体积小，但是缺点也很明显，容易出错

max模式​

max模式，即混合模式，由mysql自行决定使用那种模式

行格式选择和隔离级别有关。若使用的是innodb存储引擎，且隔离级别是rc,ru时，只能使用row模式记录

当binlog的记录模式为row模式时，对于DDL语句，仍使用基于语句模式

在5.7中默认为row模式

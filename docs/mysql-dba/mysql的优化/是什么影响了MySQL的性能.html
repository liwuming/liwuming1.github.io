mysql服务器性能优化

磁盘io是mysql的性能瓶颈

mysql在运行过程中会产生各种日志文件，error log,access log,binlog ,

错误日志， 查询日志， 慢查询日志 二进制日志--binlog

默认情况下，系统仅打开了错误日志，关闭了其他所有日志，以达到尽可能减少io损耗，提高系统性能目的；

binlog对数据安全至关重要，所以一般情况下，都会将其打开，

事务安全​

mysql事务是如何实现的

何为事务，事务就是一条或多条SQL语句组成的集合，有一种状态转化另一种状态，要么所有SQL都执行成功，要么所有SQL都执行失败；

默认情况下，事务是自动提交的，如果要执行由多条SQL组成的事务，必须使用begin..end/startraction开启一个事务；

在开启事务时，会生成一个新的lsn，日志序列号，类似git的版本号 每执行一条SQL，都事务缓冲区中会记录事务日志，

当所有SQL都执行成功时，可以发送commit命令用来提交事务，将数据改变持久化到磁盘

当所有SQL都执行成功时，可以发送rollback命令用来回滚事务，将数据改变到执行事务之出的样子

预写日志的方式来保障事务的完整性，这也就意味着是不同步的，可通过show命令查看差距 show engine innodb status;
